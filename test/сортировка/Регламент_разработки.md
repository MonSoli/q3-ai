## 1. Методология

| Роль | Ответственный |
|------|--------------|
| Product Owner | Романов А.С. |
| Scrum Master | Петрова М.С. |
| Tech Lead | Иванов А.П. |

## 2. Git Flow

- `main` — продакшн, только через PR
- `develop` — основная ветка разработки
- `feature/*` — новые фичи
- `hotfix/*` — срочные исправления

### Правила коммитов:
```
feat: добавлена авторизация через OAuth
fix: исправлена ошибка загрузки файлов >10МБ
docs: обновлена документация API
```

## 3. Code Review

- Минимум **2 аппрува** для мержа в `main`
- Автоматические проверки: ESLint, pytest, mypy
- Время на ревью: не более **24 часов**

## 4. Деплой

1. CI/CD: GitLab CI
2. Staging: автоматический деплой из `develop`
3. Production: ручной триггер после QA

## 5. Мониторинг

- **Sentry** — ошибки в реальном времени
- **Grafana** — метрики производительности
- Дежурный разработчик: ротация еженедельно
